# Contributing to [FlexiBerry UI]

First off, thank you for considering contributing to our Svelte UI project! Your help is invaluable, and we're excited to have you on board.

## Code of Conduct

This project and everyone participating in it are governed by our [Code of Conduct](CODE_OF_CONDUCT.md). By participating, you are expected to uphold this code. Please report unacceptable behavior to [email@example.com].

## How Can I Contribute?

### Reporting Bugs

If you encounter a bug, please report it by following these steps:

1. **Search for existing issues**: Ensure the bug hasn't already been reported.
2. **Create a new issue**: If it hasn't been reported, create a detailed issue with:
   - A descriptive title
   - Steps to reproduce the issue
   - Expected and actual results
   - Any relevant screenshots or logs
   - Your environment (OS, browser, Svelte version, etc.)

### Suggesting Enhancements

If you have an idea for a new feature or an improvement, we’d love to hear about it. To suggest an enhancement:

1. **Search for existing suggestions**: Make sure your idea hasn't already been suggested.
2. **Create a new issue**: Provide a clear and detailed description of your idea, including:
   - A descriptive title
   - The motivation behind the idea
   - How it would benefit the project
   - Any potential downsides or trade-offs

### Submitting Pull Requests

We welcome pull requests! To ensure a smooth process, please follow these steps:

1. **Fork the repository**: Create a personal fork of the project.
2. **Create a new branch**: Name your branch descriptively, e.g., `feature/add-new-component` or `bugfix/fix-issue-123`.
3. **Make your changes**: Ensure your code adheres to our coding standards and passes all tests.
4. **Commit your changes**: Write clear and concise commit messages. Follow the format:
   - `feat: A brief description of the feature`
   - `fix: A brief description of the bug fix`
   - `docs: A brief description of documentation changes`
5. **Push to your fork**: Push the changes to your forked repository.
6. **Submit a pull request**: Go to the original repository and submit a pull request. Include:
   - A descriptive title
   - A summary of the changes
   - Reference any related issues (e.g., `Fixes #123`)

### Development Guidelines

To ensure consistency and quality, please follow these guidelines when contributing to the codebase:

- **Code Style**: Adhere to the project's code style. We use [Prettier](https://prettier.io/) for formatting.
- **Testing**: Ensure that your code includes tests and that all tests pass before submitting a pull request.
- **Documentation**: Update the documentation to reflect your changes. This includes comments in the code, README updates, and any relevant wiki entries.

### Getting Started

1. **Clone the repository**:
   ```sh
   git clone https://github.com/flexiberry/ui.git
   cd ui
   ```

2. **Install dependencies**:
   ```sh
   npm install
   ```

3. **Start the development server**:
   ```sh
   npm run dev
   ```

4. **Run tests**:
   ```sh
   npm test
   ```
